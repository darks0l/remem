# ReMEM Context Window Benchmark — 2026-05-03

Synthetic, deterministic benchmark for testing retrieval from memory entries that are deliberately outside a simulated fixed recent-context window.

## Configuration

- Total memories: 10,000
- Query count: 250
- Simulated fixed window: 100 recent entries
- Approx corpus tokens: 720,631
- Approx fixed-window tokens: 7,214
- Corpus/window pressure: 99.9x
- Seed: 1337

## Interpretation

- Fixed-context baseline should be 0% because every query targets a record outside the simulated recent window.
- Exact-key retrieval measures whether ReMEM can recover known identifiers outside the context window.
- Natural-language retrieval without embeddings is intentionally included as a negative/limitation baseline.
- Embedding-backed retrieval measures semantic recall, but ingestion time includes local embedding generation and is hardware/model dependent.
